# Боевой движок «Arena-X»
Слушай, нам нужен прототип боевой системы для новой пошаговой RPG. Пока без графики, чисто консольный движок.

## Суть:
> Мне нужно, чтобы в игре были разные типы юнитов. На данном этапе хватит троих: Мечник, Маг и какой-нибудь Лекарь.

## Требования к функционалу:
1. **Унификация:** Я хочу иметь возможность создать «отряд» из этих ребят и одной командой заставить их всех действовать. Мне всё равно, кто там в списке — маг или мечник, у каждого должна быть кнопка «Сделать ход».
2. **Индивидуальность:** Мечник при совершении хода должен рубить мечом (выводи текст в консоль).
    - Маг должен кастовать заклинание.
    - Лекарь должен не атаковать, а восстанавливать здоровье кому-то (или просто писать, что он лечит).
3. **Живучесть:** У каждого бойца есть полоска здоровья (HP) и имя.
    - Важно: Здоровье не должно «ломаться». Если у юнита 100 HP, я не должен иметь возможности случайно прописать ему -500 или 9999 напрямую из другой части программы. Сделай так, чтобы изменение состояния персонажа шло через твои правила.
4. **Масштабируемость:** Завтра я приду и скажу: «Добавь лучника, робота и боевого кота». Твой код не должен рассыпаться. Я должен иметь возможность добавить новый тип бойца, просто создав один новый файл, не переписывая основной движок.

## Твой результат
> В финальном классе (запускаторе) создай список или массив, запихай туда разных бойцов и в одном цикле заставь их всех «проявить себя» в бою.
