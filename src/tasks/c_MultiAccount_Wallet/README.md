# Гибкий Кошелек (Multi-Account Wallet)

> **Заказчик:** «Мы делаем финтех-стартап. У пользователя может быть несколько типов счетов, и мы хотим проводить платежи максимально удобно».

### Требования:

1. **Типы счетов:**
   * **Дебетовый (обычный):** Нельзя уходить в минус. Вообще никак.
   * **Кредитный:** Можно уходить в минус до определенного лимита. Если ушел в минус — при каждом списании берется комиссия 5%.
   * **Бонусный:** С него нельзя снимать «наличные», им можно только оплачивать покупки. При каждой покупке на него капает кэшбэк (например, 1% от суммы).
2. **Унификация:** У системы должна быть функция `processPayment(Сумма)`. Ей должно быть всё равно, какой тип счета ей подсунули — она должна просто вызвать метод «Оплатить», а счет сам разберется, хватает ли денег и какая там комиссия.
3. **История:** Каждый счет должен хранить историю последних трех операций (просто текстовые строки в `history.txt`).
4. **Валидация:** Нельзя создать счет с отрицательным начальным балансом. Нельзя перевести отрицательную сумму.
